## /inbox — 요청을 문서로 저장

> **목적**: 대화 응답 대신 `docs/0-inbox/`에 구조화된 문서를 생성한다.
> **핵심**: 사용자가 말한 것을 정리해서 파일로 남기는 것이지, 질문에 답하는 것이 아니다.

## 파일명 규칙

```
{순번}-[{카테고리}]{제목}.md
```

- **순번**: `docs/0-inbox/` 내 마지막 파일 번호 + 1, 두 자리 zero-pad (01, 02, ... 10, 11)
- **카테고리**: 내용에 맞는 태그 하나 (아래 목록 참조)
- **제목**: kebab-case, 영문 소문자

### 카테고리 목록

| 태그 | 용도 |
|------|------|
| `idea` | 아이디어, 제안 |
| `backlog` | 할 일, 과제 |
| `retro` | 회고, 리뷰 |
| `explain` | 해설, 개념 정리 |
| `prd` | 요구사항 명세 |
| `vision` | 방향, 비전 |
| `decision` | 결정 사항 기록 |
| `question` | 열린 질문, 탐구 |

사용자가 카테고리를 직접 지정하면 그대로 사용. 지정하지 않으면 내용에서 판단.

## 문서 구조

```markdown
# {제목} — {YYYY-MM-DD}

## 배경

{왜 이 문서가 필요한가, 어떤 맥락에서 나왔는가}

## 내용

{사용자 요청을 구조화한 본문}

## 다음 행동

{이 문서에서 파생되는 액션이 있으면 기록. 없으면 생략}
```

## 실행 절차

1. `docs/0-inbox/` 내 기존 파일 목록을 읽어 마지막 순번을 확인한다
2. 사용자의 요청에서 카테고리와 제목을 결정한다
3. 문서를 작성하여 저장한다
4. 파일 경로를 사용자에게 알려준다

## 규칙

- 문서 생성 후 대화에서 긴 설명을 하지 않는다. 파일 경로만 알려주면 된다.
- 사용자가 추가 내용을 말하면 같은 파일에 append한다.
- 헤더의 날짜는 작성일 기준이다.
