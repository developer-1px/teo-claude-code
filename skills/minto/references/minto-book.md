# 바바라 민토 《논리의 기술》 레퍼런스

> 원제: *The Minto Pyramid Principle: Logic in Writing, Thinking and Problem Solving*

## 그루핑 라벨 테스트

같은 그룹에 묶인 아이디어가 하나의 복수 명사로 라벨링 가능한지 확인한다.
- "권고사항들", "원인들", "문제들", "변경 사항들"
- 첫 번째가 "어떤 것을 해야 하는 이유"라면, 나머지도 모두 같은 것을 해야 하는 이유여야 한다.
- 실패하면 이질적인 아이디어가 섞여 있다는 신호.

## MECE 판정

- **위반**: "비용 절감"과 "경비 삭감"은 겹친다 (상호배타 위반)
- **충족**: "비용 절감", "시간 절약", "리스크 감소"는 각각 구별된다
- MECE가 아니면 "아까 그 얘기 아니었나?" (겹침) 또는 "그건 어떻게 되는 거죠?" (빠짐)

## So What 테스트

- **지적 공백**: "다음 사항을 검토해야 한다" — 내용 없는 껍데기
- **카테고리 라벨**: "마케팅에 관한 사항들" — 요약이 아님
- **진짜 요약**: "마케팅 예산을 30% 재배분해야 한다"

## 행동 요약 vs 상황 요약

- **행동**: 실행 시 최종 결과물로 요약. "검토한다" → "비용 절감 기회 3건을 식별한다"
- **상황**: 유사성이 암시하는 추론으로 요약. "이것들이 함께 의미하는 것은 X이다"
- 행동에 상황식 요약을 붙이거나 반대를 하면 공허해진다.

## 논리적 순서 3가지

1. **시간순**: 인과 연쇄. 각 단계가 원인→결과인지 확인.
2. **구조순**: MECE 분해. 빠진 부분이 있으면 논리가 무너진다.
3. **정도순**: 중요도/크기/심각도. 기준을 먼저 명시.

## 연역 vs 귀납

| | 연역 | 귀납 |
|---|---|---|
| 구조 | 대전제→소전제→결론 | 유사 사실 그루핑→결론 |
| 장점 | 반박 어려움 | 결론 먼저 가능 |
| 단점 | 끝까지 읽어야 결론 | 그루핑 부적절하면 약함 |
| 제한 | 3단계 이내 | — |

상위 레벨은 귀납 우선. 연역 체인이 길면 독자가 전제를 잊는다.

## 문제 정의 4요소

| 요소 | 내용 |
|---|---|
| 출발점 | 기존에 잘 돌아가던 구조 |
| 방해 사건 | 문제를 일으킨 변화 |
| R1 | 원하지 않는 결과 (현재 상태) |
| R2 | 원하는 결과 (목표 상태) |

→ 도입부 변환: 출발점→S, 방해사건→C, R1/R2 gap→Q

## 7가지 문제 유형

1. R2 달성 방법을 알지만 확실하지 않다
2. 방법은 알지만 실행 방법을 모른다
3. 해결책을 실행했지만 효과가 없었다
4. 해결책이 여러 개인데 선택을 못한다
5. 문제는 알지만 원인을 모른다
6. 문제는 알지만 무엇을 해야 할지 모른다
7. 기회가 있는 것 같은데 활용법을 모른다

## 피라미드를 쓰면 안 되는 상황

- 감정적 준비가 필요한 경우 (나쁜 소식)
- 명확한 권고안이 없는 탐색 단계
- → 서사적 전개가 더 효과적

## 프레젠테이션 원칙

- 1슬라이드 = 1메시지
- 제목 = 액션 타이틀 (라벨 아님)
- 액션 타이틀만 순서대로 읽으면 스토리라인 성립
- 분석 과정이 아니라 결론을 보여준다
- 데이터가 아니라 인사이트를 보고한다
